CDR - Basic Call Count by Date Range
-- Generates a report of the number of completed calls and the total duration of calls to a given list of DNs. There are a few considerations:
-- You can add as many directory numbers to the top of the SQL query
-- The start and end date must be set in TWO places towards the bottom.
-- This counts call legs and not calls. If a call has 59,000 legs, then it MAY show up as 59,000 calls.
-- I will not swear to the absolute accuracy of the report. There are probably odd calls flows that could cause legs to be missed or counted more than once
-- Minor modifications COULD be made to count devices instead of directory numbers, which may be of greater use

WITH directory_number (dn) AS (VALUES
('5034944251'),('5034944256')
)
SELECT
directory_number.DN,
COALESCE(Activity.calls,0) AS calls,
COALESCE(Activity.duration,'00:00:00'::interval) duration
FROM directory_number
LEFT JOIN
(
-- roll up the list of DNs and durations
SELECT
ActivitySummery.DN,
COUNT(ActivitySummery.DN) AS calls,
SUM(ActivitySummery.duration) AS duration
FROM
(
-- generate a list of DNs and durations that match the critera
SELECT * FROM
(
(
-- generate a list of all calls where the calling party number matches
SELECT
callingPartyNumber AS DN,
duration
FROM
callmanager.cdr
WHERE
dateTimeOrigination BETWEEN '2020-Mar-01' AND '2020-Mar-31' AND
callingPartyNumber IN (SELECT dn from directory_number) AND
duration > '00:00:00'::interval
)
UNION ALL
(
-- generate a list of all calls where the final called party number matches
SELECT
finalCalledPartyNumber AS DN,
duration
FROM
callmanager.cdr
WHERE
dateTimeOrigination BETWEEN '2020-Mar-01' AND '2020-Mar-31' AND
finalCalledPartyNumber IN (SELECT dn from directory_number) AND
duration > '00:00:00'::interval
)
) AS ActivtyDetail
) AS ActivitySummery
GROUP BY ActivitySummery.DN
) AS Activity ON (directory_number.DN = Activity.DN)

ORDER BY DN

CDR - Finding Calls with Party that Disconnects
If you need to determine which party disconnected from the call you need to search the callmanager.cdr database in NetInfo.  Then you need to cross reference the origcallterminationonbehalfof and destcallterminationonbehalfof fields.

Database Fields for Disconnect Reason
From Cisco Call Detail Records Field Descriptions

origCallTerminationOnBehalfOf

0, Positive Integer

Specifies code that identifies why the originator was terminated.

For example, if the originator of the call hangs up the phone, the OnBehalfOf code shows "12" for Device. If the call terminates because of a transfer, the OnBehalfOf code shows "10" for Transfer.

See topics related to CDR field descriptions for a list of the codes. This release added new OnBehalfOf codes.

Default - 0

destCallTerminationOnBehalfOf

0, Positive Integer

Specifies code that identifies why the destination was terminated.

For example, if the destination of the call hangs up the phone, the OnBehalfOf code shows "12" for Device. If the call terminates because of a transfer, the OnBehalfOf code shows "10" for Transfer.

See topics related to CDR field descriptions for a list of the codes. This release added new OnBehalfOf codes.

Default - 0

CDR - Basic Call Count by Date Range


-- Generates a report of the number of completed calls and the total duration of calls to a given list of DNs. There are a few considerations:
-- You can add as many directory numbers to the top of the SQL query
-- The start and end date must be set in TWO places towards the bottom.
-- This counts call legs and not calls. If a call has 59,000 legs, then it MAY show up as 59,000 calls.
-- I will not swear to the absolute accuracy of the report. There are probably odd calls flows that could cause legs to be missed or counted more than once
-- Minor modifications COULD be made to count devices instead of directory numbers, which may be of greater use
WITH directory_number (dn)
      AS (VALUES ('5034944251'), ('5034944256'))
SELECT directory_number.dn,
        Coalesce(Activity.calls, 0)                         AS calls,
        Coalesce(Activity.duration, '00:00:00' :: interval) duration
FROM   directory_number
        LEFT JOIN (
                  -- roll up the list of DNs and durations
                  SELECT ActivitySummery.dn,
                        COUNT(ActivitySummery.dn)     AS calls,
                        Sum(ActivitySummery.duration) AS duration
                  FROM   (
                          -- generate a list of DNs and durations that match the critera
                          SELECT *
                          FROM   ((
                                  -- generate a list of all calls where the calling party number matches
                                  SELECT callingpartynumber AS DN,
                                          duration
                                    FROM   callmanager.cdr
                                    WHERE  datetimeorigination BETWEEN
                                          '2020-Mar-01' AND '2020-Mar-31'
                                          AND callingpartynumber IN (SELECT dn
                                                                      FROM
                                              directory_number)
                                          AND duration > '00:00:00' :: interval)
                                  UNION ALL
                                  (
                                  -- generate a list of all calls where the final called party number matches
                                  SELECT finalcalledpartynumber AS DN,
                                          duration
                                    FROM   callmanager.cdr
                                    WHERE  datetimeorigination BETWEEN
                                          '2020-Mar-01' AND '2020-Mar-31'
                                          AND finalcalledpartynumber IN (SELECT
                                              dn
                                                                          FROM
                                              directory_number)
                                          AND duration > '00:00:00' :: interval)
                                  ) AS
                                  ActivtyDetail) AS
                          ActivitySummery
                  GROUP  BY ActivitySummery.dn) AS Activity
              ON ( directory_number.dn = Activity.dn )
ORDER  BY dn

CDR - Finding Calls with Party that Disconnects
If you need to determine which party disconnected from the call you need to search the callmanager.cdr database in NetInfo.  Then you need to cross reference the origcallterminationonbehalfof and destcallterminationonbehalfof fields.

Database Fields for Disconnect Reason
From Cisco Call Detail Records Field Descriptions

origCallTerminationOnBehalfOf

0, Positive Integer

Specifies code that identifies why the originator was terminated.

For example, if the originator of the call hangs up the phone, the OnBehalfOf code shows "12" for Device. If the call terminates because of a transfer, the OnBehalfOf code shows "10" for Transfer.

See topics related to CDR field descriptions for a list of the codes. This release added new OnBehalfOf codes.

Default - 0

destCallTerminationOnBehalfOf

0, Positive Integer

Specifies code that identifies why the destination was terminated.

For example, if the destination of the call hangs up the phone, the OnBehalfOf code shows "12" for Device. If the call terminates because of a transfer, the OnBehalfOf code shows "10" for Transfer.

See topics related to CDR field descriptions for a list of the codes. This release added new OnBehalfOf codes.

Default - 0

Uncompleted Calls to Frontier in Last 10 Days


select
      date_trunc('hour',datetimeorigination) as interval,
      count(*),
      repeat('■',(count(*)::float / 10)::int) as bar
from
      callmanager.cdr_augmented
where
      datetimeorigination > current_date -10 and
      finalcalledpartynumber_carrier = 'FRONTIER COMMUNICATIONS NORTHWEST, INC. - OR' and
      duration = '00:00:00' and
      origdevicename not like 'RightFax-Prod-%'
group by 
      date_trunc('hour',datetimeorigination)
order by
      date_trunc('hour',datetimeorigination)
